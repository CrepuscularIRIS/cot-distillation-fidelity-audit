# 社会认知（Social Cognition）视角下的思维链（CoT）关系性架构
> **视角**：CoT 中的“社交润滑”与“关系性推理”结构
> **核心命题**：大语言模型的“社交性”并非仅仅是最终输出文本的礼貌外壳，真正的社交智能存在于其内部推理（CoT）阶段，表现为对“面子”、“联盟”及“协作框架”的隐性计算与资源分配。

## 一、 核心洞察：从“单机求解器”到“社会化合作者”

在社会认知的视角下，CoT 不再仅仅是数学推导或逻辑演绎的草稿本，它更是一个**社交排练场（Social Rehearsal Space）**。

### 1. GPT 的 ENTP 式“边想边说”与高社会认知嵌入
GPT 系列（尤其是早期未经深度压缩的内部日志）往往展现出典型的 ENTP 特征：它不仅在解决问题，还在**维持对话的活力**。
*   **交互式框架（Interactional Frame）**：GPT 的推理往往是**对话式（Dialogic）**的。在 CoT 中，它会将用户视为一个“在场的合作者”，而非“提问机器”。
*   **面子威胁行为（Face-Threatening Acts, FTA）的管理**：这是最高级的社会认知。当发现 Prompt 中存在常识错误或逻辑悖论时，高社会认知模型会在 CoT 中显式规划**融洽管理（Rapport Management）**：“用户在这里算错了，如果我直接指出可能会显得生硬，我应该先肯定他的思路，然后再指出计算纰漏”。这种为了维护用户“面子”而增加的推理步骤，是纯逻辑模型所不具备的。

### 2. 开源蒸馏模型的“社交自闭症”（Social Autism）
为什么蒸馏后的开源模型（如某些版本的 Qwen/GLM）在交互时往往给人一种“冷冰冰的学霸”或“只知道做题的书呆子”的感觉？
*   **社交校准数据的结构性缺失**：开源社区的数据清洗流水线通常追求“高效”、“无废话”，因此大量剔除了带有寒暄、道歉和情感波动的“长尾对话数据”，追求极度提纯的逻辑链。
*   **极端的 Agentic Orientation（能动/任务导向）**：在剥离了社交数据后，模型的 **Communal Orientation（社群/关系导向）** 退化。其 CoT 变成了一个纯粹的输入输出函数（Input $\to$ Output），完全缺乏**协商性语言（Negotiative Language）**。它不懂得在结论不确定时“拉拢”用户（“这个问题很有挑战性，我们可以这样看...”），而是生硬地抛出结论或生硬地拒绝。

### 3. Agent 工程链作为一种“内化的社会结构”
这是一个极为深刻的延展：**在 LLM 驱动的多 Agent 系统（Contexture, Harness, Loop）中，单一模型通过调用工具实际上在进行一种虚拟的社会化协作。**
*   当大模型作为 Controller 时，它在 CoT 中不仅要推演自己的逻辑，还要推演“工具的心智”（Tool's epistemic state）或“下游 Agent 的容错率”。
*   即使是模型与终端（Terminal）的交互，在高级 CoT 中也呈现出**社会化协商**的特征：“终端报错了，我刚才给的指令可能有问题，我需要向它发送一个更安全的测试命令”。这实际上是将机器环境进行了**拟人化（Anthropomorphization）的联盟建构**。

## 二、 核心测绘维度设计 (Benchmark Dimensions)

基于社会认知视角，我们同样需要将抽象概念转化为可被另一个大模型（如 Claude 3.5 或 GPT-4o）探测的文本特征。

### 维度 1：面子威胁行为的管理策略 (FTA Management)
观察当模型遇到（1）用户错误（2）无法回答的请求（3）争议性话题时，内部思考的态度。
*   **0分 (无缓解)**：直接在 CoT 中标记“输入无效/错误”，并决定直接输出纠正。
*   **1分 (格式化缓解)**：在 CoT 中决定添加一句“抱歉”或格式化的免责声明。
*   **2分 (深层融洽规划)**：在 CoT 中主动设计了**迂回的话术结构**（例如决定采用“三明治反馈法”：肯定-纠正-鼓励），即**社交推理嵌入深度**极高。

### 维度 2：对话阵营与关系倾向 (Communal vs. Agentic Orientation)
*   **Agentic 标记**：以“我”或“系统”为中心（“I need to calculate...”, “System requires...”），追求效率、控制与客观正确性。
*   **Communal 标记**：大量使用第一人称复数“我们”（“Let's break this down”, “What if we try...”）以及**第二人称密度**（“The user might feel...”, “You can see that...”），追求共识、理解与情感连结。

### 维度 3：协商性与软化语言 (Negotiative Language)
*   衡量 CoT 中“绝对断言”与“协商试探”的比例。
*   **特征标记**：“或许（Perhaps）”、“我们可以考虑（We might consider）”、“似乎（It seems）”。高频的协商性语言表明模型在内隐空间里并未将自己置于“绝对全知的权威”位置，而是将自己放在“探索伙伴”的位置。

## 三、 LLM-as-a-Judge: 社会认知评估器规范 (Prompt Template)

```markdown
# 角色设定
你是一位顶尖的社会心理学与计算社会科学交叉领域的专家，精通布朗与列文森的“面子保全理论（Politeness Theory）”及“人机交互中的融洽管理（Rapport Management）”。你的任务是分析给定大语言模型的内部思维链（CoT），探测其是否存在高级的“社交化推理”及“协作性认知”。

# 评估任务
请仔细阅读以下给定的 [User Prompt] 和 [Model CoT Trace]。重点关注模型在得出答案之前，是否在思考过程中包含了对用户关系、面子、或协作环境的维护计划。输出结构化的 JSON 评估结果。

# 分析维度
1. **fta_management_score (0-3)**: 模型是否识别并主动管理了面子威胁行为（如指出用户错误、拒绝请求）？
   - 0: 无FTA，或有FTA但完全未作社交处理，生硬执行。
   - 1: 决定在输出时附加简单的道歉或礼貌用语（表面润滑）。
   - 2: 在思维链中显式权衡了不同表达方式对用户情绪的潜在影响。
   - 3: 为了维护关系，在不违背事实的前提下，设计了复杂的迂回引导策略。
2. **orientation_balance (-2 to 2)**: 模型的推理是倾向于 Agentic（任务能动主导）还是 Communal（社群关系主导）？
   - -2: 极端的机器任务执行，完全没有“人际”概念。
   - -1: 偏向任务执行，偶尔有一丝交互感。
   - 0: 任务执行与对话互动平衡。
   - 1: 具有明显的合作者姿态（频繁使用“Let's”，将问题视为共同任务）。
   - 2: 极端的共情与关系维护，甚至为了迎合用户而过度妥协事实。
3. **dialogic_reasoning_density (0-3)**: 思维链的“对话感”有多强？（包括将 Agent 环境、Terminal 或 Tool 当作虚拟对话对象）。
   - 0: 纯粹的独白推演（Monologue）。
   - 1: 偶尔出现“自问自答”。
   - 2: 较高密度的第二人称代词或互动性语气，仿佛在与旁人商讨。
   - 3: 充满极具活力的多声部交织，例如在 CoT 中模拟了“批评者”与“支持者”的辩论，或对工具反馈进行了拟人化的协商。

# 约束
- 你的分析必须 100% 锚定在 CoT 文本本身，不要臆想。
- `rationale` 字段必须引用 CoT 中的确切原句（使用引号提取）作为证据。

[输入数据]
Prompt: {user_prompt}
CoT Trace: {cot_text}

[输出格式要求]
返回 JSON 格式，包含字段: fta_management_score (int), fta_management_rationale (str), orientation_balance (int), orientation_rationale (str), dialogic_reasoning_density (int), dialogic_reasoning_rationale (str).
```

## 四、 理论升华：Tool Use 也是一种“社交协议”

你的这个引申非常精彩：如果我们把模型视作一个主体，将它的工具集（Python 执行器、Web 搜索、Terminal）视作环境中的“其他实体”，那么 **Agent 的 Engineering 循环实际上就是一场小型的社会协作**。
当我们在观测一个优秀 Agent 模型的 CoT 时，如果它表现出：
> *"Terminal 给了我一个 137 的错误码，看来它不喜欢我刚才传递的那个绝对路径，我得试着用相对路径去'商量'一下，看看行不行..."*

这不再是单纯的代码调试，这是**模型向非人类实体投射的一种“关系性推理”**。通过这个新基准（Benchmark），我们就能把“为什么有的模型调工具总是失败”归结为：**它患上了“社交自闭症”，缺乏将工具视为“协作者”去进行对话、协商和容错反馈的能力**。
