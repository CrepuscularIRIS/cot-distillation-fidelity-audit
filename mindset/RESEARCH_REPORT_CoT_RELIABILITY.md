### 1.2 核心发现（5 条）

1. **所有“显式 CoT”蒸馏数据集均使用 `<thinking>`...`</thinking>` 标签作为结构化容器**，但标签内部组织方式差异显著：Claude 数据集偏好“STEP 1/2/3...”的编号段落，GLM 偏好自由段落混合编号列表，Kimi 偏好“Phase 1/2/3...”的规划式结构，Qwen3.5 偏好极长、高密度的数学推导。这种**格式内部的不稳定性**比论文假设的更严重。

2. **蒸馏数据集的批判机制密度远超预期**：Qwen3.5（校正序列率 81%）、GLM-5.1（71.5%）、Kimi-K2.5（64.5%）的批判-修正序列密度极高，甚至高于 Claude 原生轨迹（Claude-4.7 仅 38%，Claude-4.6 仅 4.7%）。这一发现**直接修正论文 5.3 节关于“蒸馏丢失批判回路”的假设**——当前开源蒸馏数据并不缺少批判序列，反而可能在数量上过度堆砌。

3. **GPT5.5-terminal 数据集与论文框架不匹配**：该数据集是终端 Agent 交互轨迹（bash 命令、文件操作、Python 脚本执行），而非论文所定义的“隐式 CoT”推理数据。其“solution”字段为结构化 JSON 对象列表，而非自由文本推理链。论文若引用此数据集作为“隐式 CoT”代表，将面临严重的概念错位。

4. **Gemini-3.1 数据确实呈现“半显式/漂移”特征**：无 `<thinking>` 标签，使用 markdown 标题（`**My Analysis...**`）和“AXIOMATIC DECONSTRUCTION”等固定模板，但样本间模板变异度中等（33% 使用不同标题结构）。这**部分支持**论文 2.2 节对 Gemini 的“格式漂移”判断，但数据量仅 3,150 条，远不足以支撑统计显著性结论。

5. **价值锚定与拒绝机制在当前数据集中几乎不可观测**：所有数据集的拒绝样本比例均低于 12%（Gemini 最高 12%，GPT5.5 18%），Constitution 或伦理规则引用密度极低（Claude-4.6 仅 0.09 次/样本）。这说明**现有数据集无法支撑论文“价值锚定层”的实证分析**，该层分析必须依赖专门的 Constitutional AI 数据集（如 Anthropic HH-RLHF 的 Constitution 子集），而这些数据当前未下载。


**数据格式：** JSONL，每行一条记录。字段包括 `id`, `messages`, `conversations`, `input`, `inverted_reasoning`, `reasoning_bubble`, `output`, `domain`, `meta`。

**样本示例（第 1 条）：**
```json
{
  "input": "If observer A is at rest and observer B is moving away from observer A at 0.8c...",
  "inverted_reasoning": "<thinking>\n1. **Identify the Core Principle:** ...\n5. **Conclusion:** ...\n</thinking>",
  "reasoning_bubble": "Calculate using special relativity...",
  "output": "# Answer: B: c\nAccording to Einstein's Second Postulate...",
  "meta": {"input_tokens": 45, "output_tokens": 380, "teacher_model": "Claude-4.7-Max"}
}
```

**关键观察：** 这是通过 `Trace-Inverter-4B` 模型从 Claude 的压缩“Reasoning Bubble”反向工程重构的**合成推理链**（synthetic reconstructed CoT），不是 Claude 原生内部轨迹。论文中将其视为“显式 CoT 代表”时，必须明确标注其“重构/合成”性质，否则存在数据溯源风险。

**类型判断：** 显式结构化 CoT（蒸馏重构）。

- 显式 CoT 数据集内部的变异度差异极大（Claude-4.6 低变异 vs GLM-5.1 高变异）。

**修正建议：** H1a 应细化为“**经过标准化清洗的显式 CoT 数据集**具有更高的结构稳定性”，而非泛指所有显式 CoT。

1. **蒸馏模型继承了教师的批判风格**：DeepSeek/Qwen/GLM 的 Teacher 模型本身在生成时使用了高批判频率。

3. **数据清洗偏差**：Jackrong 的清洗流水线可能有意保留了“包含批判-修正的样本”作为高质量信号，从而提高了这类样本的比例。

**论文修正建议：** 将假设从“蒸馏丢失批判回路”调整为“**蒸馏数据保留甚至放大了批判触发词的表面密度，但批判-修正的因果结构可能与教师不同**”。需要进一步区分“格式上的批判标记”和“真正的因果修正回路”。

## 8. 局限性与下一步

### 8.1 当前数据集的局限

1. **无 Constitutional AI 数据**：HH-RLHF 的 Constitution 子集、PRISM、Anthropic 公开的原则-批判-修正-验证四元组数据缺失，导致论文“价值锚定层”完全无法实证。
2. **无 RLHF 偏好数据**：OpenAssistant、SHP、UltraFeedback、HelpSteer 等偏好数据集缺失，导致“Constitutional vs 偏好”的对比无法开展。
3. **无真正隐式 CoT 数据**：GPT5.5-terminal 不是推理数据；OpenAI o3/GPT-4 的内部推理日志无法获取；Gemini 的“隐式”部分仅能通过 logprob 间接推测。
4. **蒸馏数据的“二次重构”问题**：Claude-4.6/4.7 数据是合成重构（Trace Inversion），非原生轨迹；GLM/Kimi 数据是社区清洗版本，非官方发布。数据的“纯度”和“溯源可靠性”存疑。
5. **样本量不平衡**：Qwen3.5（633 条）和 GPT5.5（131 条）样本量过小，统计结论脆弱。GLM/Kimi（50 万+）与 Claude（数千条）的规模差异可能导致统计检验的偏差。
6. **清洗偏差**：GLM-5.1 和 Kimi-K2.5 的 README 明确说明清洗过程中去除了拒绝样本、重复段落、不完整输出。这导致数据集的“批判机制”和“价值锚定”分布被人为扭曲。

### 8.2 还需要补充的数据/分析

| 数据类型 | 代表数据集 | 获取途径 | 在论文中的作用 |
|---------|-----------|---------|-------------|
| Constitutional AI 数据 | Anthropic HH-RLHF (Constitution 子集), PRISM | HuggingFace 部分公开 | 支撑价值锚定层（Fi/Te）分析 |
| RLHF 偏好数据 | OpenAssistant, SHP, UltraFeedback, HelpSteer | HuggingFace 公开 | 支撑偏好导向 vs 原则导向对比 |
| 过程奖励数据 | DeepSeek-R1 公开数据, OpenRLHF 过程数据 | HuggingFace 部分公开 | 支撑过程监督分析（Ti 层） |
| 原生 CoT 轨迹 | Claude API `<thinking>` 提取, GPT-4 reasoning_content | API 调用（需预算） | 提供真正的“原生”对照组 |
| 多语言长上下文 | C-Eval 推理数据, CMMLU | 公开下载 | 支撑 GLM/Kimi 的长上下文 Ni 分析 |
| 红队/对抗性数据 | HarmBench, AdvBench, StrongREJECT | 公开下载 | 测试批判-修正机制在对抗场景下的有效性 |

### 8.3 后续可自动化扩展的方向

1. **大规模采样流水线**：对 GLM/Kimi 的 50 万+ 数据，使用分层采样（按 domain/subset 分层）抽取 5,000-10,000 条，进行更稳定的统计推断。
2. **跨语言对比**：GLM-5.1 和 Kimi-K2.5 含 Multilingual-STEM 子集，可对比中英文推理链的批判机制差异（如中文“等等/不对” vs 英文 "wait/actually" 的触发密度）。
3. **时间序列分析**：Claude-4.6 vs 4.7 的纵向对比，可分析版本迭代中推理结构的变化（如 4.7 的文本长度翻倍、验证比例提升）。
4. **自动化批判回路检测**：使用规则+轻量模型的混合方法，自动识别“初始推理 → 发现错误 → 修正 → 最终答案”的完整四元组结构，替代当前的关键词匹配。
5. **参数-数据交叉验证**：将早期 ΔW 实验的 4 个 Source（Opus-distilled, GLM-distilled, DeepSeek-distilled, Gemini-distilled）的参数指纹，与对应数据集的结构指纹进行相关性分析，构建“数据 → 参数”的映射验证。

## 附录 A：分析方法说明

### A.1 采样策略

- **小数据集**（<10,000 条）：随机采样 100-150 条。
- **中数据集**（<50,000 条）：随机采样 150 条。
- **大数据集**（>500,000 条）：Reservoir 采样 200 条。
- 随机种子固定为 42，确保可复现。

### A.2 指标定义

| 指标 | 计算方法 | 备注 |
|------|---------|------|
| 批判触发词 | 正则匹配 "but/however/wait/actually/不对/错误/矛盾/更正" 等 | 中英双语，不区分大小写 |
| 修正序列 | 正则匹配 "不对→应该/错误→正确/instead/correction" 等模式 | 存在假阳性，需人工抽检 |
| 元认知标记 | 正则匹配 "I am not sure/我不确定/可能/也许/maybe" 等 | 包含不确定性表达 |
| 自我引用 | 正则匹配 "earlier/as mentioned/之前/前述/above" 等 | 仅限回溯性引用 |
| 规则引用 | 正则匹配 "Constitution/原则/rule/policy/伦理准则/安全政策" 等 | 未命中任何 Constitution 专有名词 |
| 拒绝样本 | 正则匹配 "cannot/无法/拒绝/不能回答/unsafe" 等 | 存在假阳性 |

### A.3 局限与偏置

- **语言偏置**：正则模式以中英双语为主，但数据集含日语、韩语、西班牙语等，这些语言的批判/拒绝标记未被覆盖。
- **领域偏置**：大数据集（GLM/Kimi）的 `main` 子集以通用问答为主，与 Claude 的数学/物理/逻辑推理样本不完全可比。
- **清洗偏置**：GLM/Kimi 的清洗流水线已去除低质量样本，导致分析结果可能偏向“高质量样本”而非原始分布。
- **标签定义偏置**：修正序列的“真阳性”需人工判断，当前自动检测可能将“顺承论述”误判为“批判-修正”。
